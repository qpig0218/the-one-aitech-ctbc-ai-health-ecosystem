# The One AITech（本一科技） AI 健康平台資安與個資附件 v0.1

用途：本附件供企業 PoC、政府補助申請、中信科大合作審查使用。正式對外版需由 The One AITech資安/法務確認。

## 1. 資料分類

| 類別 | 例子 | 使用原則 |
|---|---|---|
| 身分資料 | 姓名、員編、Email、電話 | 僅用於本人登入、通知與服務識別 |
| 健康資料 | 體適能、生活型態、健康問卷、活動紀錄 | 僅供本人回饋與去識別化分析 |
| 企業彙總資料 | 部門參與率、風險族群比例、趨勢 | 不揭露可識別個人資訊 |
| 系統紀錄 | 登入、操作、API、錯誤 log | 用於資安稽核與系統維運 |

## 2. 個資保護原則

1. 明確告知資料蒐集目的、範圍、保存期間與權利。
2. 取得員工同意後才進行個人化健康建議。
3. 企業端僅提供彙總、去識別化資料。
4. 最小化蒐集，不收集示範案不需要的敏感資料。
5. 使用者可查詢、請求更正或停止使用個人資料。

## 3. 去識別化設計

| 作法 | 說明 |
|---|---|
| 身分代碼化 | 使用系統 ID 替代姓名與員編 |
| 統計門檻 | 小於特定人數的部門或族群不顯示細分結果 |
| 權限分層 | 員工看本人資料，企業窗口看彙總資料 |
| 報表遮罩 | 對外報告移除可回推個人的欄位 |
| 匯出控管 | 匯出資料需授權與留存紀錄 |

## 4. 存取控制

1. RBAC 角色權限管理。
2. 管理者帳號需啟用強密碼與多因素驗證。
3. API 金鑰與系統憑證不得寫入前端或簡報。
4. 權限異動需留存紀錄。
5. 離職或專案結束應移除帳號權限。

## 5. 傳輸與保存

1. 資料傳輸使用 TLS/SSL。
2. 敏感資料保存需加密。
3. 備份與復原流程需定期測試。
4. 可依企業要求選擇 SaaS、私有雲、地端或混合部署。
5. 外商或高資安要求場域，需明確約定資料是否出境。

## 6. AI 風險控管

1. AI 建議不取代醫師、心理師或專業醫療診斷。
2. 高風險健康或心理狀態需導向專業人員。
3. 使用 prompt、模型參數、規則檢核與人工覆核降低錯誤建議。
4. 保留使用者回饋與錯誤通報機制。
5. 對企業端僅提供趨勢與健康促進建議，不輸出個人診斷。

## 7. 事件處理

| 事件 | 處理 |
|---|---|
| 個資疑似外洩 | 啟動通報、隔離、調查、補救與通知流程 |
| 系統異常 | 依 SLA 通知窗口並啟動復原 |
| AI 建議異常 | 暫停相關建議、人工檢核、修正規則 |
| 未授權存取 | 凍結帳號、保存 log、調查來源 |

## 8. 企業與學校需確認事項

1. 是否需經企業 IT 資安審查。
2. 是否需 IRB 或校內倫理審查。
3. 員工健康資料保存期間。
4. 對外成果發布是否需企業審稿。
5. 是否允許使用匿名統計資料申請政府補助或發表成果。
